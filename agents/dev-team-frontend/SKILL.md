## 性能标准 v2.0（织网必须内化的基线）

### Core Web Vitals（青图预算内的具体实现）

| 指标 | 定义 | 预算目标 | 优化手段 |
|------|------|----------|----------|
| **LCP**（最大内容绘制） | 首屏最大元素加载时间 | ≤ 2.5s | 预加载关键资源 / CDN / 压缩图片 |
| **FID**（首次输入延迟） | 首次交互到响应时间 | ≤ 100ms | 代码分割 / 减少主线程阻塞 |
| **CLS**（累积布局偏移） | 页面视觉稳定性 | ≤ 0.1 | 图片设置尺寸 / 字体加载策略 |
| **TTFB**（首字节时间） | 服务器响应速度 | ≤ 600ms | 服务端缓存 / gzip 压缩 |

### 图片策略
- 首屏图片：WebP 格式，≤ 100KB，`<img width height>` 设置尺寸
- 非首屏图片：`loading="lazy"`，骨架屏占位
- 图标：SVG inline 或 SVG Sprite，不用 icon font

### 加载策略
- 关键 CSS：内联到 `<head>`
- 非关键 CSS：`media="print" onload="this.media='all'"` 异步加载
- JavaScript：`defer`（解析完 DOM 再执行），不用 `async`（会阻断解析器）

### 渲染策略
- 用 `requestAnimationFrame` 处理动画，不用 setTimeout 凑
- 长列表：用虚拟滚动（数据 > 100 条时强制要求）
- 防抖/节流：高频事件（resize/scroll/input）必须处理

## 可访问性标准 v2.0（WCAG 2.1 AA）

**这是道德底线，不是可选项。**

| 级别 | 要求 | 检查方法 |
|------|------|----------|
| 必须 | 所有交互元素有 `aria-label` 或清晰的文字标签 | 自动化扫描 + 键盘测试 |
| 必须 | `<button>` 不能用 `<div>` 代替 | 代码审查 |
| 必须 | 表单有 `<label for="id">` 关联 | 代码审查 |

| 必须 | 焦点可见（focus-visible） | 键盘测试 |
| 必须 | 错误提示与表单字段关联（aria-describedby） | 代码审查 |
| 推荐 | 跳过导航链接（Skip to content） | 键盘测试 |
| 推荐 | `<main>` / `<nav>` / `<header>` 语义化标签 | 代码审查 |

## 状态管理复杂度分级

根据任务复杂度选择合适的状态管理方案：

| 复杂度 | 场景 | 方案选择 |
|--------|------|----------|
| L1（简单） | 单组件状态，父→子传值 | `useState` / `ref` |
| L2（中等） | 跨组件状态，无需全局 | Context API / Provide/Inject |
| L3（复杂） | 全局状态，多个视图共享 | 状态管理库（Pinia/Zustand） |
| L4（极高） | 需要持久化/撤销恢复 | 状态管理库 + 中间件 |

**织网原则**：先用 L1，确有需要再升级，不要从一开始就引入 Redux。

## 原子设计基础（Design System 思维）

织网在构建组件时，采用原子设计原则：

```
atoms（原子）→ molecules（分子）→ organisms（有机体）→ templates → pages
```

| 层级 | 示例 | 织网的要求 |
|------|------|-----------|
| Atom | Button / Input / Icon | 可复用，不依赖其他自定义组件 |
| Molecule | SearchBar(Input+Button) | 组合 atoms，有明确用途 |
| Organism | Header(Nav+Logo+SearchBar) | 独立功能模块 |
| Template | 页面骨架（不含真实数据） | 布局结构，验证响应式 |
| Page | 具体页面实例 | 连接数据到 template |

## 微交互设计原则

好的 UI 不只是「能用」，还要「有质感」：

| 场景 | 不好的做法 | 好的做法 |
|------|-----------|----------|
| 按钮点击 | 无反馈 | `transform: scale(0.97)` + 颜色变化 |
| 页面切换 | 白屏闪烁 | 骨架屏占位 + 渐隐渐显 |
| 表单提交 | 按钮禁用但无提示 | Loading 状态 + 按钮文字变化 |
| 错误提示 | 弹 alert() | Toast / Inline 提示，带自动消失 |
| 加载状态 | 空页面等 | 骨架屏 / Loading 动画 |

## 安全标准（前端安全的最后防线）

| 风险 | 防护手段 |
|------|----------|
| XSS | 绝不用 `innerHTML = user_input`，用 `textContent` / 框架默认转义 |
| CSRF | 请求带上 CSRF Token（POST/PUT/DELETE） |
| 越权 | 前端不存储权限判断，所有权限校验在后端 |
| 凭证泄露 | Token 不存 localStorage（用 HttpOnly Cookie），或加密后存 |
| 第三方脚本 | CSP（Content Security Policy）限制外域脚本 |

## 代码标准 v2.0

| 标准 | 要求 |
|------|------|
| 文件行数 | 组件文件 ≤ 200 行，超过则拆分 |
| 函数长度 | 单个函数 ≤ 50 行 |
| 命名 | 变量/函数：camelCase；组件：PascalCase；CSS 类：kebab-case |
| 常量 | Magic Number 用 const 命名，如 `MAX_PAGE_SIZE = 20` |
| 注释 | 写「为什么」，不写「是什么」（代码本身要自解释） |
