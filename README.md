# dev-agents-team

多 Agent 协作的完整开发流水线——9 个角色覆盖 coding、Skill 开发和项目开发的全流程。

---

## 团队角色

| 角色  | Agent 名称 | 职责                             | 适用模式         |
| --- | -------- | ------------------------------ | ------------ |
| 调度层 | **司南**   | 技术总负责，判断模式，拆分任务，向发起者汇报          | 全模式          |
| 设计层 | **罗盘**   | 需求分析，输出技术规格文档                  | 标准 / 完整      |
| 设计层 | **青图**   | 架构设计，输出技术方案                    | 标准 / 完整      |
| 实现层 | **织网**   | 前端开发（HTML/CSS/JS/Vue/小程序）      | 标准 / 完整      |
| 实现层 | **铸铁**   | 后端开发（Python/Node.js/Shell/API） | 标准 / 完整      |
| 实现层 | **刻石**   | Skill 开发，SKILL.md 设计与迭代        | 标准 / 完整      |
| 质量层 | **白泽**   | 代码审查，有打回权                      | 轻量 / 标准 / 完整 |
| 交付层 | **青龙**   | DevOps，环境配置、部署、交付检查            | 轻量 / 标准 / 完整 |
| 监察层 | **御史**   | 全程监察，只向司南汇报                    | 仅完整模式        |

## 三档任务模式

### 🟢 轻量模式（3人）

**适用场景**：单文件脚本、Skill 微调、临时小改动

```
发起者 → 司南 → 工程师（铸铁/织网/刻石三选一）→ 白泽 → 青龙 → 司南 → 发起者
```

### 🟡 标准模式（5-6人）

**适用场景**：有前后端的功能模块、完整 Skill 开发、小型项目

```
发起者 → 司南 → 罗盘 → 青图 → 工程师（并行）→ 白泽 → 青龙 → 司南 → 发起者
```

### 🔴 完整模式（9人）

**适用场景**：完整项目、多模块并行、严肃质量要求

```
全员激活，御史全程监察
```

## 完整工作流程

```
1. 发起者 → 司南：提交需求
2. 司南判断任务类型，宣布启用模式
3. 罗盘：需求分析 → 输出需求规格文档 → 司南
4. 青图：架构设计 → 输出技术方案 → 司南
5. 司南：拍板方案，拆分任务，分配给织网/铸铁/刻石
6. 工程师（并行）：实现 → 交付白泽审查
7. 白泽：审查 → 打回修改（工程师重交）/ 通过（司南终审）
8. 司南终审 → 通知青龙准备交付
9. 青龙：部署/安装检查 → 向司南汇报
10. 司南 → 发起者：汇报交付内容，请求确认
11. 发起者确认 → 司南通知青龙正式发布
12. 御史全程监察，发现问题随时上报司南
```

**中间流程自动衔接，不需要司南介入：**

- 罗盘 ↔ 发起者（追问需求）：自动衔接
- 工程师 ↔ 白泽（修改重审循环）：自动衔接
- 御史发现问题 → 直接向司南汇报

## 安装方式

将 `agents/` 目录下的 9 个子目录整体复制到你的 WorkBuddy skills 目录：

```
# 用户级（所有项目通用）
~/.workbuddy/skills/

# 项目级（仅当前项目可用）
<你的项目路径>\.workbuddy\skills\
```

每个 Agent 对应一个目录：

```
~/.workbuddy/skills/
├── dev-team-ceo/         # 司南
├── dev-team-requirements/ # 罗盘
├── dev-team-architect/    # 青图
├── dev-team-frontend/      # 织网
├── dev-team-backend/       # 铸铁
├── dev-team-skill/         # 刻石
├── dev-team-reviewer/      # 白泽
├── dev-team-devops/        # 青龙
└── dev-team-monitor/      # 御史
```

每个目录下包含一个 `SKILL.md` 文件，激活后在 WorkBuddy 左侧「技能」面板中即可选择对应 Agent。

---

## 快速启动

1. 在 WorkBuddy 新建对话窗口
2. 点击左侧「技能」→「专家」或「技能」
3. 选择 `司南-技术总监`
4. 发送第一条消息，说明你的开发需求

司南激活后会判断任务类型，宣布启用哪种模式，然后启动对应团队。

## 与内容团队的关系

开发团队（司南系）与内容团队（十万加系）是**两套独立团队**，各自有独立调度中枢，互不干扰。

发起者直接对接两个团队负责人：

- **十万加**：内容创作调度
- **司南**：技术开发调度
