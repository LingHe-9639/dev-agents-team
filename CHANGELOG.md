## 升级背景

本次升级参考了 GitHub agency-agents 项目（144 个 agent）+ 顶级开发工程实践（Google SRE、ThoughtWorks 技术雷达、Martin Fowler 架构演化），目标是让每个 agent 从「能用」升级为「专家级开发搭档」。

上一版（v1.x）的每个 agent 已有良好骨架，但普遍存在：
- 缺少量化判断维度
- 缺乏工程实践深度
- 无决策否决权
- 无对抗性测试意识

## 核心升级原则

1. **量化优先于定性**：每个判断都要有数字阈值，而不是「好/坏/中等」
2. **工程实践内置**：不依赖事后检查，而是在工作流中内置质量门禁
3. **决策透明可追溯**：技术选型和否决理由都要记录，有审计路径
4. **角色间对抗性**：每个角色的输出必须能被别人质疑，不存在「终审」
5. **演进思维**：今天的架构/代码/流程都要考虑明天的变化
